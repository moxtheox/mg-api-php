FeedObserver creates a detached Fiber that the Reactor never tracks:

🟡 Medium

Add a MultiCallBatch wrapper — Geotab's API supports ExecuteMultiCall natively, which reduces round trips. Your waitAll is clever but it still fires N HTTP requests. A batch builder could serialize them into one:

php

The next one is the (string)$content ?? '' null-coalescing issue in Reactor::await():

php
return (string)$content ?? '';
The null-coalescing operator here is unreachable dead code — casting anything to (string) first means it can never be null, so the ?? '' never fires. A null $content would just become an empty string from the cast alone.

What you actually want is to coalesce before the cast:

php
return (string)($content ?? '');
It's a subtle but meaningful difference — the intent is "if content is null, use an empty string, then cast." The current code reads as "cast content to string, then if that result is null use empty string" which is logically impossible.

Good approach but there's a timing problem. The HTTP status check is happening before Fiber::suspend() — at that point curl_multi_exec has just been called once but the request almost certainly hasn't completed yet. The response code will be 0 for an in-flight request, so your >= 400 check will never fire when it should.

The status check needs to happen after the fiber resumes, meaning after the request has actually completed:

php
public function await(mixed $ch): string {
    $fiber = Fiber::getCurrent() ?? throw new RuntimeException("Async ops must run in a Fiber.");
    
    curl_multi_add_handle($this->mh, $ch);
    $this->map[(int)$ch] = $fiber;

    $active = 0;
    curl_multi_exec($this->mh, $active);
    
    $url = curl_getinfo($ch, CURLINFO_EFFECTIVE_URL);
    $this->hostMap[(int)$ch] = parse_url($url, PHP_URL_HOST) ?: 'default';
    
    Fiber::suspend();

    // Request is complete at this point
    $code = curl_getinfo($ch, CURLINFO_HTTP_CODE);
    if ($code >= 400) {
        throw HTTPResponseException::fromStatusCode($code, $url);
    }

    $content = curl_multi_getcontent($ch);
    curl_multi_remove_handle($this->mh, $ch);
    return (string)($content ?? '');
}
Also note $curlInf['code'] isn't a valid curl_getinfo key — the correct constant is CURLINFO_HTTP_CODE. Using the named constants instead of array keys is more reliable and readable. So "0000000000000000" is the sentinel value that tells Geotab to start from the very beginning of the feed rather than the current position. That's different from omitting fromVersion entirely — omitting it anchors to now, while "0000000000000000" starts from the oldest available data.

So we actually have three distinct starting states:

<?php
declare(strict_types=1);

namespace Geotab\Services;

use Geotab\Client;
use Geotab\Core\Reactor;


class FeedObserver {
    private ?string $currentVersion;

    public function __construct(
        private Client $client,
        private string $type,
        private ?string $modelClass = null,
        ?string $fromVersion = null
    ) {
        $this->currentVersion = $fromVersion;
    }

    public ?string $fromVersion {
        get => $this->currentVersion;
    }

    public function start(
        callable $onData,
        ?array $search = null,
        ?int $resultsLimit = 50000
    ): void {
        $reactor = Reactor::getInstance();
        $reactor->register(function() use ($reactor, $onData, $search, $resultsLimit) {
            while (!$reactor->stopping) {
                $params = [
                    'typeName'     => $this->type,
                    'resultsLimit' => $resultsLimit
                ];

                if ($this->currentVersion !== null) {
                    $params['fromVersion'] = $this->currentVersion;
                }

                if ($search !== null) {
                    $params['search'] = $search;
                }

                $start = microtime(true);

                // HTTP503ResponseException will propagate up and terminate the observer.
                // The consumer is responsible for catching it and restarting with a backoff.
                $res = $this->client->call(
                    method: 'GetFeed',
                    params: $params,
                    modelClass: $this->modelClass
                );

                // Update version immediately after response is received
                $this->currentVersion = $res->toVersion;

                if ($res->count > 0) {
                    $onData($res->data);
                }

                // Adaptive throttling — enforce 1 call/second floor, back off when feed is quiet
                $elapsed = microtime(true) - $start;

                if ($res->count === $resultsLimit) {
                    // Full payload: data is saturated, stay aggressive but respect the floor
                    if ($elapsed < 1.0) {
                        $this->client->wait(1.0 - $elapsed);
                    }
                } elseif ($res->count < $resultsLimit / 2) {
                    // Light payload: feed is quiet, long backoff
                    $this->client->wait(30.0);
                } else {
                    // Moderate payload: feed is active but not saturated, medium backoff
                    $this->client->wait(15.0);
                }
            }
        });
    }
}

  },
});

  sendResult?.results?.[0]?.message_group_id ||
  sendResult?.results?.[0]?.message_group?.id;
Step 3: Poll for Results
Ace processes queries asynchronously. You have to poll. Give it 10 seconds before your first poll, then retry every 8 seconds. In my experience 6 attempts is enough.
JavaScript

await sleep(10000);
for (let attempt = 0; attempt < 6; attempt++) {
  const pollResult = await geotab.call('GetAceResults', {
    serviceName: 'dna-planet-orchestration',
    functionName: 'get-message-group',
    customerData: true,
    functionParameters: { message_group_id: messageGroupId },

  const messageGroup = pollResult?.results?.[0]?.message_group;

  if (status === 'DONE') {

    for (const key of Object.keys(messages)) {

      // msg.reasoning contains the natural language insight
