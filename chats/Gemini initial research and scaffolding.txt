Introduced in PHP 8.1, Fibers are the foundation for modern async PHP. However, unlike JS async, Fibers do not provide an event loop or non-blocking I/O out of the box. They are "stackful coroutines" that allow you to pause and resume code execution.
+1

Crucial Distinction: In JS, await returns control to the event loop. In PHP, Fiber::suspend() returns control to whatever started the Fiber (usually a scheduler you have to provide or pull from a library).

A. Swoole (The "Node.js on Steroids" Extension)

Since you’re looking at this from an architectural perspective, the best way to understand Fibers (introduced in PHP 8.1) is to see them as low-level primitives for cooperative multitasking.

Unlike JavaScript's async/await, which is a syntax-level feature that "colors" your functions, Fibers are a runtime capability that allows you to pause and resume a call stack at any point.

1. The Mechanics: Stackful Coroutines
In JavaScript, an async function returns a Promise. To get the value, every function in the chain must also be async and use await.

PHP Fibers are stackful. This means:

You can suspend execution deep inside a nested function call.

The parent functions don't need to know they are being suspended.

The entire call stack (local variables, return addresses) is preserved in memory and swapped out.

2. The "No Color" Advantage
As a Principal Engineer, you've likely dealt with the "Function Coloring" problem in Node.js (where sync and async code don't mix easily). Fibers solve this.

JS: If getData() becomes async, main() must become async.

PHP: getData() can call Fiber::suspend(). main() remains a standard function. It has no idea the stack was paused and swapped out while waiting for I/O.

3. The Lifecycle of a Fiber
A Fiber exists in one of several states. You control it manually via the Fiber class:

Method	Description
start()	Begins execution of the Fiber.
suspend(value)	Pauses the Fiber and returns value to the caller.
resume(value)	Restarts the Fiber, sending value back into the suspension point.
getReturn()	Gets the final return value once the Fiber finishes.
Basic Code Example
PHP
$fiber = new Fiber(function (): void {
    echo "Fiber started...\n";
    $value = Fiber::suspend('interrupted'); 
    echo "Fiber resumed with: $value\n";
});

$status = $fiber->start(); // Output: Fiber started...
echo "Main thread received: $status\n"; // Output: Main thread received: interrupted

$fiber->resume('secret_key'); // Output: Fiber resumed with: secret_key
4. The "Catch": Fibers are not an Event Loop
This is the most common point of confusion for engineers coming from Node.js or Go.

Fibers do not make your I/O non-blocking.

If you call file_get_contents() inside a Fiber, the entire PHP process still blocks. To achieve true concurrency, you need:

A Scheduler: A loop that manages which Fiber to resume and when.

Non-blocking I/O: Using stream_select() or extensions like ev or uv to check if a socket is ready without blocking the thread.

5. Architectural Implementation: Revolt
Because writing a scheduler is tedious, the PHP community converged on Revolt. It acts as the "Event Loop" for PHP.

When you use a modern async library (like Amphp), it uses Revolt to manage Fibers. When you perform an I/O operation:

The library calls Fiber::suspend().

Control returns to the Revolt loop.

The loop handles other tasks (or other Fibers).

Once the socket is ready, Revolt calls $fiber->resume().

Summary for Decision Making
Performance: Fibers have very low overhead compared to OS threads.

Complexity: They are a "power user" feature. Most of your developers should use a framework (like Amp or ReactPHP) rather than interacting with the Fiber class directly.

Use Case: Ideal for high-concurrency I/O (proxies, chat servers, scrapers) without the "callback hell" or "promise chaining" of older PHP versions.

<?php

class AsyncKernel {
    private array $fibers = [];

    public function submit(Fiber $fiber): void {
        $this->fibers[] = $fiber;
        $fiber->start();
    }

    public function run(): void {
        while (!empty($this->fibers)) {
            foreach ($this->fibers as $i => $fiber) {
                if ($fiber->isTerminated()) {
                    unset($this->fibers[$i]);
                    continue;
                }

                // If the fiber is suspended, we try to resume it
                // In a real app, you'd check if the specific I/O is ready here
                if ($fiber->isSuspended()) {
                    $fiber->resume();
                }
            }
            
            // "Keep other stuff working" - Simulate background tasks
            echo "[Kernel] Doing background maintenance...\n";
            usleep(500000); // 0.5s heartbeat
        }
    }
}

/**
 * A Non-blocking Curl Mock using Fibers
 */
function async_curl_get(string $url) {
    $ch = curl_init($url);
    curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
    curl_setopt($ch, CURLOPT_NONBLOCKING, true);

    $mh = curl_multi_init();
    curl_multi_add_handle($mh, $ch);

    echo "[Fiber] Starting request to $url\n";

    // The Magic Loop
    do {
        $status = curl_multi_exec($mh, $active);
        
        // If the network is busy, we YIELD control back to the kernel
        if ($active) {
            echo "[Fiber] I/O busy, suspending...\n";
            Fiber::suspend(); 
        }
    } while ($active && $status === CURLM_OK);

    $response = curl_multi_getcontent($ch);
    
    curl_multi_remove_handle($mh, $ch);
    curl_multi_close($mh);
    
    return "Response from $url (Length: " . strlen($response) . ")";
}

// --- Execution ---

// Create a Fiber for the "Fetch"
$fetchFiber = new Fiber(function() {
    $result = async_curl_get("https://www.google.com");
    echo "[Fiber] Finished! Data: $result\n";
});

$kernel->submit($fetchFiber);
$kernel->run();

1. The Core Reactor (The "Engine")
This class manages the lifecycle of all concurrent requests and resumes the correct Fibers when data returns.

PHP
<?php
declare(strict_types=1);

namespace Geotab\Core;

use Fiber;
use RuntimeException;

final class Reactor {
    private static ?self $instance = null;
    private mixed $mh;
    /** @var array<int, Fiber> Mapping resource ID to Fiber */
    private array $map = [];

    private function __construct() {
        $this->mh = curl_multi_init();
    }

    public static function getInstance(): self {
        return self::$instance ??= new self();
    }

    /**
     * Registers a curl handle and suspends the current Fiber.
     */
    public function register(mixed $ch): string {
        $fiber = Fiber::getCurrent() ?? throw new RuntimeException("Must run in a Fiber");
        
        curl_multi_add_handle($this->mh, $ch);
        $this->map[(int)$ch] = $fiber;

        // PAUSE: Return control to the loop in run()
        Fiber::suspend();

        $result = curl_multi_getcontent($ch);
        curl_multi_remove_handle($this->mh, $ch);
        curl_close($ch);
        
        return $result;
    }

    public function run(): void {
        $active = 0;
        do {
            // Check for activity on the sockets
            curl_multi_exec($this->mh, $active);

            // Handle completed requests
            while ($info = curl_multi_info_read($this->mh)) {
                $ch = $info['handle'];
                $id = (int)$ch;
                
                if (isset($this->map[$id])) {
                    $fiber = $this->map[$id];
                    unset($this->map[$id]);
                    // RESUME: Jump back into the SDK method
                    $fiber->resume();
                }
            }

            // Efficiency: Don't busy-wait. 
            // curl_multi_select blocks until there is network activity.
            if ($active > 0) {
                curl_multi_select($this->mh, 0.1);
            }
        } while ($active > 0 || !empty($this->map));
    }
}

class GeotabFeed {

    /**
     * @template T
     * @param class-string<T> $class
     * @param callable(array<T>): void $onData
     */
    public function watch(string $typeName, string $class, string $fromVersion, callable $onData): void {
        new Fiber(function() use ($typeName, $class, $fromVersion, $onData) {
            $currentVersion = $fromVersion;

            while (true) {
                // 1. Trigger the async fetch for the next batch
                // This will suspend the Fiber while the Reactor handles the I/O
                $result = $this->client->get('GetFeed', $class, [
                    'typeName' => $typeName,
                    'fromVersion' => $currentVersion
                ]);

                // 2. Update the version pointer immediately
                $currentVersion = $result->toVersion;

                // 3. Before we even process the data, we COULD trigger 
                // the next fetch here if we wanted deeper pipelining.
                // But even a simple loop like this is non-blocking to the REST of the app.
                
                if ($result->count > 0) {
                    $onData($result->data);
                } else {
                    // Geotab Feed best practice: if no data, back off for a few seconds
                    // We need a non-blocking sleep!
                    $this->asyncSleep(5); 
                }
            }
        })->start();
    }

    private function asyncSleep(int $seconds): void {
        $stopAt = microtime(true) + $seconds;
        // Suspend until the Reactor sees that time has passed
        while (microtime(true) < $stopAt) {
            Fiber::suspend();
        }
    }
}

/**
 * @template T
 */
class BatchRunner {
    private const MAX_CHUNK_SIZE = 50;

    public function __construct(private GeotabClient $client) {}

    /**
     * @param array<RequestDTO> $requests
     * @return array<CallResult>
     */
    public function execute(array $requests): array {
        // 1. Chunk the work
        $chunks = array_chunk($requests, self::MAX_CHUNK_SIZE);
        $finalResults = [];
        $fibers = [];

        foreach ($chunks as $chunkIndex => $chunk) {
            $fibers[] = new Fiber(function() use ($chunk, &$finalResults, $chunkIndex) {
                try {
                    // 2. Use the native ExecuteMultiCall for this chunk
                    $batchResult = $this->client->nativeMultiCall($chunk);
                    
                    // 3. Map results back to their original indices
                    foreach ($batchResult as $itemIndex => $data) {
                        $originalKey = ($chunkIndex * self::MAX_CHUNK_SIZE) + $itemIndex;
                        $finalResults[$originalKey] = new CallResult(result: $data);
                    }
                } catch (Throwable $e) {
                    // 4. If the chunk fails, we mark only THIS chunk as failed
                    foreach ($chunk as $itemIndex => $_) {
                        $originalKey = ($chunkIndex * self::MAX_CHUNK_SIZE) + $itemIndex;
                        $finalResults[$originalKey] = new CallResult(error: $e);
                    }
                }
            });
            
            $fibers[array_key_last($fibers)]->start();
        }

        // 5. Let the Reactor drive the I/O for all chunks simultaneously
        Reactor::getInstance()->run();
        
        ksort($finalResults); // Ensure order is preserved
        return $finalResults;
    }
}

class GeotabFeed {
    public function __construct(
        private GeotabClient $client,
        private int $resultsLimit = 50000
    ) {}

    public function watch(
        string $typeName, 
        string $class, 
        string $fromVersion, 
        callable $onData,
        int $pollIntervalSeconds = 5 // User-controlled rate
    ): void {
        new Fiber(function() use ($typeName, $class, $fromVersion, $onData, $pollIntervalSeconds) {
            $currentVersion = $fromVersion;

            while (true) {
                try {
                    $result = $this->client->get('GetFeed', $class, [
                        'typeName' => $typeName,
                        'fromVersion' => $currentVersion,
                        'resultsLimit' => $this->resultsLimit
                    ]);

                    $currentVersion = $result->toVersion;

                    if ($result->count > 0) {
                        $onData($result->data);
                    }

                    // --- The Back-off Logic ---
                    // If we got less than the limit, we've caught up. 
                    // Wait for the user-defined interval.
                    if ($result->count < $this->resultsLimit) {
                        $this->asyncSleep($pollIntervalSeconds);
                    }
                    // If count == limit, the loop continues immediately 
                    // (Fiber resumes in the next Reactor tick).

                } catch (GeotabRateLimitException $e) {
                    // Handle 429: Explicit back-off regardless of payload size
                    $this->asyncSleep(30); 
                }
            }
        })->start();
    }

    private function asyncSleep(int $seconds): void {
        $target = microtime(true) + $seconds;
        // This is a "Cooperative Sleep"
        // It lets other Fibers (other Feeds or Batch calls) run 
        // while this specific Feed Fiber waits.
        while (microtime(true) < $target) {
            Fiber::suspend();
        }
    }
}

1. The Core Reactor (src/Core/Reactor.php)
This is the heart of the system. It uses curl_multi to multiplex I/O and manages Fiber suspension/resumption.

PHP
<?php
declare(strict_types=1);

namespace Geotab\Core;

use Fiber;
use RuntimeException;

/**
 * The Reactor acts as the Event Loop for the SDK.
 * It manages non-blocking I/O using curl_multi.
 */
final class Reactor {
    private static ?self $instance = null;
    private mixed $mh;
    private array $map = []; // Handle ID -> Fiber

    private function __construct() {
        $this->mh = curl_multi_init();
    }

    public static function getInstance(): self {
        return self::$instance ??= new self();
    }

    /**
     * Registers a curl handle and yields control of the current Fiber.
     */
    public function await(mixed $ch): string {
        $fiber = Fiber::getCurrent() ?? throw new RuntimeException("Async ops must run in a Fiber.");
        
        curl_multi_add_handle($this->mh, $ch);
        $this->map[(int)$ch] = $fiber;

        Fiber::suspend();

        $content = curl_multi_getcontent($ch);
        curl_multi_remove_handle($this->mh, $ch);
        return (string)$content;
    }

    public function run(): void {
        $active = 0;
        do {
            curl_multi_exec($this->mh, $active);

            while ($info = curl_multi_info_read($this->mh)) {
                $id = (int)$info['handle'];
                if (isset($this->map[$id])) {
                    $fiber = $this->map[$id];
                    unset($this->map[$id]);
                    $fiber->resume();
                }
            }

            if ($active > 0) {
                // Blocks until I/O activity or timeout, preventing CPU spin
                curl_multi_select($this->mh, 0.1);
            } elseif (!empty($this->map)) {
                // Handles suspended fibers that might be waiting on timers, not I/O
                usleep(1000); 
            }
        } while ($active > 0 || !empty($this->map));
    }
}

The Resilient Multi-Call Aggregator (src/Services/Batcher.php)

<?php

namespace Geotab\Services;

use Fiber;
use Throwable;
use Geotab\Client;
use Geotab\Models\Response;

/**
 * Solves the "Atomic Failure" issue of native ExecuteMultiCall.
 */
class Batcher {
    private const CHUNK_SIZE = 50;

    public function __construct(private Client $client) {}

    /**
     * @param array<array{method: string, params: array}> $calls
     * @return array<int, mixed|Throwable>
     */
    public function execute(array $calls): array {
        $chunks = array_chunk($calls, self::CHUNK_SIZE, true);
        $results = [];
        $fibers = [];

        foreach ($chunks as $chunk) {
            $fiber = new Fiber(function() use ($chunk, &$results) {
                try {
                    // We call a specialized 'ExecuteMultiCall' endpoint
                    $response = $this->client->call('ExecuteMultiCall', [
                        'calls' => $chunk
                    ], \stdClass::class);

                    foreach ($chunk as $index => $originalCall) {
                        // Map results back to original keys
                        $results[$index] = $response->data[$index] ?? null;
                    }
                } catch (Throwable $e) {
                    // If the chunk fails, only these indices get the error
                    foreach ($chunk as $index => $_) {
                        $results[$index] = $e;
                    }
                }
            });

            $fiber->start();
            $fibers[] = $fiber;
        }

        // The Reactor must be running to process the I/O for these fibers
        // Reactor::getInstance()->run(); // Usually called by the entry point

        return $results;
    }
}

<?php
declare(strict_types=1);

namespace Geotab\Models;

/** @template T */
abstract class Response {
    public function __construct(
        public readonly array $data,
        public readonly ?string $toVersion = null
    ) {}

    public int $count { get => count($this->data); }

    /**
     * Factory method to handle specific hydration logic for the type
     */
    abstract public static function fromArray(array $json): static;
}

/** @extends Response<object> */
class LoginResponse extends Response {
    public static function fromArray(array $json): static {
        $result = $json['result'] ?? throw new \RuntimeException("No login data");
        // Login is unique; it doesn't always behave like an array of entities
        return new static([$result]);
    }

    public function getSessionId(): string {
        return $this->data[0]['credentials']['sessionId'] 
            ?? $this->data[0]['sessionId'] 
            ?? throw new \RuntimeException("Missing sessionId");
    }
}

/** @template T; @extends Response<T> */
class EntityResponse extends Response {
    public static function fromArray(array $json, string $modelClass = \stdClass::class): static {
        $results = $json['result'] ?? [];
        $toVersion = $json['toVersion'] ?? null;

        $models = array_map(function($item) use ($modelClass) {
            // Polymorphic check: Does the model handle its own hydration?
            if (is_subclass_of($modelClass, Hydratable::class)) {
                return $modelClass::fromArray($item);
            }
            // Fallback for stdClass or simple DTOs
            return (object)$item;
        }, $results);

        return new static($models, $toVersion);
    }
}

// src/Core/Reactor.php

public function run(): void {
    while ($this->hasWork()) {
        $this->step();
    }
}

public function hasWork(): bool {
    return !empty($this->map) || !empty($this->timers);
}

public function step(): void {
    $active = 0;
    curl_multi_exec($this->mh, $active);

    while ($info = curl_multi_info_read($this->mh)) {
        $id = (int)$info['handle'];
        if (isset($this->map[$id])) {
            $fiber = $this->map[$id];
            unset($this->map[$id]);
            $fiber->resume();
        }
    }

    $now = microtime(true);
    foreach ($this->timers as $key => $timer) {
        if ($now >= $timer['at']) {
            $fiber = $timer['fiber'];
            unset($this->timers[$key]);
            $fiber->resume();
        }
    }

    if ($active > 0) {
        curl_multi_select($this->mh, 0.01); // Very fast tick
    }
}
2. Updated Client.php (Direct Execution)
We wrap the internal logic in a Fiber automatically. If we aren't already inside a run() loop, we "pulse" the reactor until our specific result is ready.

// src/Client.php

public function call(
    string $method, 
    array $params, 
    string $responseClass = EntityResponse::class,
    ?string $modelClass = null,
    bool $isAuthCall = false
): Response {
    $result = null;
    $error = null;

    $fiber = new \Fiber(function() use ($method, $params, $responseClass, $modelClass, $isAuthCall, &$result, &$error) {
        try {
            $ch = curl_init("https://{$this->currentServer}/apiv1");
            
            $rpcParams = $params;
            if (!$isAuthCall) {
                if (!$this->sessionId) throw new RuntimeException("Not authenticated.");
                $rpcParams['credentials'] = ['database' => $this->database, 'sessionId' => $this->sessionId];
            }

            $body = json_encode(['method' => $method, 'params' => $rpcParams]);
            curl_setopt_array($ch, [
                CURLOPT_POST => true,
                CURLOPT_POSTFIELDS => $body,
                CURLOPT_RETURNTRANSFER => true,
                CURLOPT_HTTPHEADER => ['Content-Type: application/json'],
                CURLOPT_TIMEOUT => 30, 
            ]);

            $rawData = Reactor::getInstance()->await($ch);
            $json = json_decode($rawData, true);
            $result = $this->processResponse($json, $responseClass, $modelClass);
        } catch (\Throwable $e) {
            $error = $e;
        }
    });

    $fiber->start();

    // If we are already in a managed loop (e.g. inside another Fiber), 
    // we don't need to do anything; the Reactor will resume us.
    // If we are in the main thread, we pulse the reactor until THIS fiber finishes.
    if (!\Fiber::getCurrent()) {
        $reactor = Reactor::getInstance();
        while (!$fiber->isTerminated()) {
            $reactor->step();
        }
    }

    if ($error) throw $error;
    return $result;
}
